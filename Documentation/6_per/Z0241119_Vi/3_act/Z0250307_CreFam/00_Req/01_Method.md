## **1️⃣ 明确需求**
需求分析要从**用户群体、核心功能、业务目标**等方面入手。

- **教师**：发布课程、批改作业、管理学生。  
- **管理员**：管理用户、课程、数据分析。  

### **📌 主要功能**
- **课程管理**：在线课程、章节结构、视频/文档/练习。  
- **学习进度**：学生学习情况、作业、考试记录。  
- **互动功能**：讨论区、问答、教师点评。  
- **用户管理**：学生、教师、管理员权限区分。  
- **支付系统**（如适用）：课程购买、会员订阅。  

### **📌 技术选型**
- **前端**：Vue.js / React  
- **后端**：Node.js / Django / Spring Boot  
- **数据库**：MySQL / PostgreSQL

## **4️⃣ 设计数据库**
数据库设计要满足**数据完整性、查询效率、扩展性**。  

### **📌 主要数据表**
1. **用户表（users）**
   ```sql
   CREATE TABLE users (
       id INT PRIMARY KEY AUTO_INCREMENT,
       username VARCHAR(50) UNIQUE,
       email VARCHAR(100) UNIQUE,
       password_hash VARCHAR(255),
       role ENUM('student', 'teacher', 'admin'),
       created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
   );
   ```
2. **课程表（courses）**
   ```sql
   CREATE TABLE courses (
       id INT PRIMARY KEY AUTO_INCREMENT,
       title VARCHAR(100),
       description TEXT,
       teacher_id INT,
       price DECIMAL(10,2),
       created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
       FOREIGN KEY (teacher_id) REFERENCES users(id)
   );
   ```
3. **章节表（chapters）**
   ```sql
   CREATE TABLE chapters (
       id INT PRIMARY KEY AUTO_INCREMENT,
       course_id INT,
       title VARCHAR(100),
       content TEXT,
       video_url VARCHAR(255),
       FOREIGN KEY (course_id) REFERENCES courses(id)
   );
   ```
4. **作业表（assignments）**
   ```sql
   CREATE TABLE assignments (
       id INT PRIMARY KEY AUTO_INCREMENT,
       course_id INT,
       student_id INT,
       submission TEXT,
       grade INT DEFAULT NULL,
       FOREIGN KEY (course_id) REFERENCES courses(id),
       FOREIGN KEY (student_id) REFERENCES users(id)
   );
   ```
5. **订单表（orders）**
   ```sql
   CREATE TABLE orders (
       id INT PRIMARY KEY AUTO_INCREMENT,
       user_id INT,
       course_id INT,
       amount DECIMAL(10,2),
       status ENUM('pending', 'paid', 'canceled'),
       created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
       FOREIGN KEY (user_id) REFERENCES users(id),
       FOREIGN KEY (course_id) REFERENCES courses(id)
   );
   ```

- **适用于：中大型团队，有长期维护计划。**  

### **🔹 看板管理（Kanban）**
- 使用 Trello / Jira 进行任务拆解。  
- **适用于：小团队、快速迭代项目。**

- **代码管理**：GitHub / GitLab / Bitbucket  
- **CI/CD**：自动化部署，减少手动干预  
- **日志监控**：Prometheus + Grafana / ELK

## **📌 如何列出所有核心功能**  

在需求分析阶段，列出所有核心功能时，需要遵循**清晰、完整、层次分明**的原则，以确保**开发、设计、测试团队都能理解和执行**。  

### **1️⃣ 关键方法**
1. **用户视角**（基于用户需求）：列出所有用户可能需要的功能。  
2. **业务视角**（基于业务需求）：列出实现商业目标的功能，如课程管理、支付等。  
3. **技术视角**（基于系统实现）：列出后台管理、API、数据库相关功能。  
4. **分层架构**：划分**核心功能 → 子功能**，让结构更清晰。

最好的功能描述句型是：  

**「用户可以 [执行某个操作] 以 [达成某个目的]。」**  

✅ **示例**：  
- **用户可以** 通过邮箱或社交账号注册 **以** 创建个人账户。  
- **用户可以** 在课程页面播放视频 **以** 进行在线学习。  
- **管理员可以** 添加、编辑课程信息 **以** 管理课程内容。  
- **用户可以** 在讨论区发帖和评论 **以** 交流学习心得。  
- **系统会** 发送通知 **以** 提醒用户即将开始的直播课程。  

这种句型**清晰、简洁、目标明确**，适用于文档、开发沟通和需求说明。
